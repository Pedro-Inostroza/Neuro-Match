// BUGS ENCONTRADOS Y CORREGIDOS

BUG #1 — FUNCIONAL: sin bloqueo del tablero

El codigo original no bloqueaba el tablero mientras se resolvia un par. Con clicks rapidos se podian voltear 3 cartas a la vez, acumulando mas de 2 indices en el arreglo volteadas[]. Ademas, hacer doble click en la misma carta la hacia coincidir consigo misma porque a === b y los emojis eran iguales.

FIX: se agrego let bloqueado = false como variable de estado. Se activa al detectar el segundo click y se desactiva al final del setTimeout. Se agregaron guardias: if (bloqueado) return y if (volteadas.includes(i)) return.

BUG #2 — ARQUITECTURA: listener por carta y lectura desde el DOM

El render original creaba un addEventListener dentro del bucle de cada carta, generando N listeners nuevos en cada reinicio (fuga de memoria). Ademas, la comparacion leia div.textContent del DOM en vez del estado, violando la regla de fuente unica de verdad.

FIX: un unico tablero.addEventListener('click', ...) con delegacion de eventos. La comparacion ahora lee desde el estado: cartas[ia].emoji === cartas[ib].emoji.

BUG #3 — SEGURIDAD: XSS con innerHTML

El mensaje de victoria usaba innerHTML para insertar texto que podia venir del usuario. Si el nombre contenia <script>alert(1)</script> u otro payload, se ejecutaba en el navegador (vulnerabilidad XSS).

FIX: reemplazado por mensajeEl.textContent = ... en todo el codigo. textContent trata el contenido como texto plano, nunca como HTML.
